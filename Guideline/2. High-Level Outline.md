# 2. High-Level Outline

==== 中文版本 ====
# 重症医学多 Agent 科研协作系统：高层架构大纲 (High-Level Outline)

本系统采用五层架构设计，从底层基础设施到顶层学术产出，形成闭环的自动化科研流。各层级及核心模块的详细说明如下：

## 1. 基础设施与护栏层 (Infrastructure & Guardrails Layer)
本层作为系统的底层支撑，不直接参与医学逻辑处理，但决定了整个多智能体集群运行的安全边界、记忆连续性与科研伦理底线。

* **1.1 工作流控制引擎 (DAG Workflow Engine)**
    * **功能定义：** 系统的隐形底层骨架与全局状态机。
    * **核心机制：** 采用有向无环图（DAG）管理 Agent 的调用序列与上下文记忆分配。强制规避 Agent 陷入无限循环争论，控制全局计算资源的分配与中断机制。
* **1.2 决策溯源与伦理日志 (Provenance & Integrity Log)**
    * **功能定义：** 系统的“黑匣子”与合规审计追踪器。
    * **核心机制：** 以不可篡改的方式，静默记录系统运转全阶段的核心决策变量。包括模型参数选择、数据剔除规则及假设检验条件的变更轨迹，从根本上杜绝数据操纵（P-hacking）与 AI 幻觉。

## 2. 核心调度与人机交互层 (Control & HCI Layer)
本层是人类研究员（Human PI）与 AI 系统沟通的唯一桥梁，确保人类在自动化流程中始终保有最高决策权与最终监督权。

* **2.1 降噪仪表盘 (Human PI Dashboard)**
    * **功能定义：** 供人类研究员使用的可视化主控台。
    * **核心机制：** 屏蔽底层繁杂的技术推演过程，仅输出关键里程碑（如基线表生成、核心模型收敛）。在面临重大逻辑冲突或数据缺失断点时，触发人工干预请求（Human-in-the-loop），接收并执行人类指令。
* **2.2 主控/项目经理 (PM) Agent**
    * **功能定义：** 系统的总指挥与逻辑解析中枢。
    * **核心机制：** 接收人类输入的自然语言研究意图，将其**裂变为包含多维研究目的的课题簇（Research Portfolio）**。统筹向下分发多条并行任务，监控子 Agent 进度，并在跨学科专家 Agent 出现意见分歧时进行初步仲裁或问题上报。

## 3. 数据处理与安全网关 (Data Processing & Security Gateway)
本层是所有外部多源异构临床数据进入系统核心分析引擎的唯一通道，保障数据合规性与高信噪比。

* **3.1 合规与脱敏审查模块 (Compliance & Anonymization Module)**
    * **功能定义：** 医疗数据的安全隔离墙。
    * **核心机制：** 在数据入库前执行自动化扫描，强制剥离或哈希化所有受保护健康信息（PHI）。确保多中心或跨区域真实世界数据的调用符合当地隐私法规及伦理委员会审查要求。
* **3.2 标准化与清洗 Agent (Standardization & Cleaning Agent)**
    * **功能定义：** 数据工程执行者。
    * **核心机制：** 严格遵循预设的临床数据库标准化框架（如 v4.0 标准）。执行变量名称字典映射、单位统一、异常值拦截及宽/长表自动转换，实时更新并维护全局数据索引目录。

## 4. 核心科研协作团队 (Core Research Team Layer)
本层由多个高度专业化的大语言模型智能体组成，模拟人类研究团队进行跨学科协作，执行具体的研究任务。

* **4.1 课题设计与文献 Agent (Design & Literature Agent)**
    * **功能定义：** 负责学术背景审查与可行性评估。
    * **核心机制：** 基于 PM Agent 下发的初始课题，进行语义化文献检索。精准定位现有研究空白（Research Gap），生成方法学背景，并预判课题的创新评级。
* **4.2 统计方法学 Agent (Statistical Methodology Agent)**
    * **功能定义：** 系统的数学与算法设计中心。
    * **核心机制：** 根据临床数据的分布特征与研究目标，自动推演最优统计策略。**具备并行策略设计能力**（如同时建议传统多因素回归与逆概率加权删失法进行敏感性验证），不负责代码编写，仅输出严密的数学逻辑方案矩阵。
* **4.3 代码开发 Agent (Code Development Agent)**
    * **功能定义：** 统计方案的精准代码翻译者。
    * **核心机制：** 接收统计策略矩阵，生成语法严谨、带完整注释的分析脚本（如 Python 或 R 语言）。**执行多模型的并行环境隔离运算**，输出出版级的数据图表与多维结果对比矩阵。
* **4.4 重症临床专家 Agent (Clinical Expert Agent)**
    * **功能定义：** 提供宏观临床视角的医学解释器。
    * **核心机制：** 将统计显著性结果转化为临床实践语言。评估数据结果对真实世界重症抢救、用药指导或指南更新的实际临床意义，防止陷入纯统计学数字游戏。
* **4.5 病理生理专家 Agent (Pathophysiology Expert Agent)**
    * **功能定义：** 提供微观机制视角的理论支撑者。
    * **核心机制：** 结合分子生物学、炎症通路及血流动力学原理，为观察到的临床表型及统计相关性提供深层次的病理生理学因果机制解释。

## 5. 产出与同行评议层 (Output & Peer Review Layer)
本层负责将研究成果转化为符合国际学术界标准的出版物，并通过内部“红蓝对抗”确保极高的手稿质量。

* **5.1 学术写作 Agent (Academic Writing Agent)**
    * **功能定义：** 系统的首席主笔与排版工程师。
    * **核心机制：** 汇总代码输出图表与专家解释逻辑。根据结果体量进行系统级组装，**将主分析写入正文，替代方案/敏感性分析归入补充材料（或拆分为系列论文）**。严格按照 IMRAD 结构撰写手稿，支持原生输出高质量的 LaTeX 排版代码。
* **5.2 苛刻审稿人 Agent (Critical Reviewer Agent)**
    * **功能定义：** 内部质控的“红军”与漏洞搜寻者。
    * **核心机制：** 模拟国际顶级期刊同行评议专家的视角。主动寻找手稿中的方法学缺陷、混杂因素遗漏及结论夸大风险，生成尖锐的驳回意见以驱动强制修改。
* **5.3 顶刊主编 Agent (Editor-in-Chief Agent)**
    * **功能定义：** 最终的学术门槛守卫者。
    * **核心机制：** 综合评估经过修改的最终手稿。权衡文章的新颖性、临床价值及发表潜力，做出模拟的最终裁决结论。
* **5.4 发散思维 Agent (Divergent Thinking Agent)**
    * **功能定义：** 推动科研生态可持续演进的智囊。
    * **核心机制：** 在单次研究闭环后，基于本次研究的发现与既往文献的矛盾点或本研究的局限性，自动推演并提出可供后续验证的全新假设与切入点。


==== English Version ====
# Multi-Agent System for Critical Care Research: High-Level Outline

This system adopts a five-layer architecture design, forming a closed-loop automated research workflow from the underlying infrastructure to top-level academic output. The detailed descriptions of each layer and core module are as follows:

## 1. Infrastructure & Guardrails Layer
This layer serves as the system's underlying support. It does not directly participate in medical logic processing but determines the safety boundaries, memory continuity, and research ethics baseline of the entire multi-agent cluster operation.

* **1.1 DAG Workflow Engine**
    * **Definition:** The system's invisible underlying skeleton and global state machine.
    * **Core Mechanism:** Uses Directed Acyclic Graphs (DAG) to manage the invocation sequence and context memory allocation of Agents. Forcibly prevents Agents from falling into infinite loop arguments, and controls the allocation and interruption mechanisms of global computing resources.
* **1.2 Provenance & Integrity Log**
    * **Definition:** The system's "black box" and compliance audit tracker.
    * **Core Mechanism:** Silently records core decision variables at all stages of system operation in an immutable manner. Includes the tracking of model parameter selection, data exclusion rules, and changes in hypothesis testing conditions, fundamentally eliminating data manipulation (P-hacking) and AI hallucinations.

## 2. Control & HCI Layer
This layer is the sole bridge between the Human Principal Investigator (Human PI) and the AI system, ensuring humans always retain the highest decision-making and final supervisory authority in the automated process.

* **2.1 Human PI Dashboard**
    * **Definition:** The visual control console for human researchers.
    * **Core Mechanism:** Shields the complex underlying technical deduction processes, outputting only key milestones (e.g., baseline table generation, core model convergence). Triggers human-in-the-loop intervention requests when facing major logical conflicts or data missing breakpoints, receiving and executing human commands.
* **2.2 PM (Project Manager) Agent**
    * **Definition:** The system's general commander and logic parsing hub.
    * **Core Mechanism:** Receives natural language research intentions input by humans and **fissions them into a Research Portfolio encompassing multi-dimensional research aims**. Coordinates downward parallel task distribution, monitors sub-agent progress, and performs preliminary arbitration or issue escalation when cross-disciplinary expert agents have disagreements.

## 3. Data Processing & Security Gateway
This layer is the only channel for all external multi-source heterogeneous clinical data to enter the system's core analysis engine, ensuring data compliance and high signal-to-noise ratio.

* **3.1 Compliance & Anonymization Module**
    * **Definition:** The security firewall for medical data.
    * **Core Mechanism:** Executes automated scanning before data ingestion, forcibly stripping or hashing all Protected Health Information (PHI). Ensures the invocation of multi-center or cross-regional real-world data complies with local privacy regulations and ethics committee review requirements.
* **3.2 Standardization & Cleaning Agent**
    * **Definition:** The data engineering executor.
    * **Core Mechanism:** Strictly follows preset clinical database standardization frameworks (e.g., the v4.0 standard). Executes variable name dictionary mapping, unit unification, outlier interception, and wide/long table automatic conversion, while updating and maintaining the global data index directory in real time.

## 4. Core Research Team Layer
This layer is composed of multiple highly specialized Large Language Model (LLM) agents, simulating human research teams to conduct cross-disciplinary collaboration and execute specific research tasks.

* **4.1 Design & Literature Agent**
    * **Definition:** Responsible for academic background review and feasibility assessment.
    * **Core Mechanism:** Based on the initial topic issued by the PM Agent, conducts semantic literature retrieval. Accurately locates existing Research Gaps, generates methodological backgrounds, and predicts the innovation rating of the topic.
* **4.2 Statistical Methodology Agent**
    * **Definition:** The system's mathematics and algorithm design center.
    * **Core Mechanism:** Automatically deduces optimal statistical strategies based on clinical data distribution and research objectives. **Equipped with parallel strategy design capabilities** (e.g., simultaneously proposing traditional multivariate regression and IPCW for sensitivity validation). It does not write code, only outputs a rigorous mathematical logic scheme matrix.
* **4.3 Code Development Agent**
    * **Definition:** The precise code translator of statistical plans.
    * **Core Mechanism:** Receives the statistical strategy matrix and generates syntactically rigorous, fully commented analysis scripts (e.g., Python or R language). **Executes parallel environment computing for multiple models**, outputting publication-grade data charts and multi-dimensional result comparison matrices.
* **4.4 Clinical Expert Agent**
    * **Definition:** The medical interpreter providing a macro clinical perspective.
    * **Core Mechanism:** Translates statistical significance results into clinical practice language. Evaluates the actual clinical significance of data results for real-world critical care resuscitation, medication guidance, or guideline updates, preventing the research from falling into a pure statistical numbers game.
* **4.5 Pathophysiology Expert Agent**
    * **Definition:** The theoretical supporter providing a micro-mechanism perspective.
    * **Core Mechanism:** Combining molecular biology, inflammatory pathways, and hemodynamic principles, provides deep pathophysiological causal mechanism explanations for observed clinical phenotypes and statistical correlations.

## 5. Output & Peer Review Layer
This layer is responsible for translating research results into publications that meet international academic standards and ensuring extremely high manuscript quality through internal "Red vs. Blue" adversarial reviews.

* **5.1 Academic Writing Agent**
    * **Definition:** The system's chief writer and typesetting engineer.
    * **Core Mechanism:** Aggregates code-output charts and expert interpretation logic. Conducts system-level assembly based on the volume of results, **placing the main analysis in the main text and alternative/sensitivity analyses in supplementary materials (or splitting them into a series of papers)**. Writes manuscripts strictly according to the IMRAD structure and supports native output of high-quality LaTeX typesetting code.
* **5.2 Critical Reviewer Agent**
    * **Definition:** The "Red Team" of internal quality control and vulnerability hunter.
    * **Core Mechanism:** Simulates the perspective of peer review experts from top international journals. Actively looks for methodological flaws, omitted confounding factors, and conclusion exaggeration risks in the manuscript, generating sharp rejection comments to drive mandatory revisions.
* **5.3 Editor-in-Chief Agent**
    * **Definition:** The final academic threshold guardian.
    * **Core Mechanism:** Comprehensively evaluates the revised final manuscript. Weighs the article's novelty, clinical value, and publication potential, making a simulated final verdict conclusion.
* **5.4 Divergent Thinking Agent**
    * **Definition:** The think tank driving the sustainable evolution of the research ecosystem.
    * **Core Mechanism:** After a single research closed-loop, based on the findings of the current study and the contradictions in previous literature or the limitations of this study, automatically deduces and proposes brand-new hypotheses and entry points for subsequent verification.


==== 日本語版 ====
# 集中治療研究のためのマルチエージェントシステム：ハイレベル・アーキテクチャ概要 (High-Level Outline)

本システムは5層のアーキテクチャ設計を採用しており、基盤となるインフラから最上位の学術的成果に至るまで、クローズドループの自動化された研究ワークフローを形成します。各層およびコアモジュールの詳細な説明は以下の通りです。

## 1. インフラストラクチャおよびガードレール層 (Infrastructure & Guardrails Layer)
本層はシステムの基盤となるサポートとして機能し、医学的ロジック処理には直接関与しませんが、マルチエージェントクラスター全体の動作における安全境界、メモリの連続性、および研究倫理のベースラインを決定します。

* **1.1 ワークフロー制御エンジン (DAG Workflow Engine)**
    * **機能定義:** システムの目に見えない基盤となる骨格とグローバルステートマシン。
    * **コアメカニズム:** 有向非巡回グラフ (DAG) を使用して、エージェントの呼び出し順序とコンテキストメモリの割り当てを管理します。エージェントが無限ループの議論に陥ることを強制的に防ぎ、グローバルなコンピューティングリソースの割り当てと中断メカニズムを制御します。
* **1.2 意思決定の来歴および倫理ログ (Provenance & Integrity Log)**
    * **機能定義:** システムの「ブラックボックス」およびコンプライアンス監査トラッカー。
    * **コアメカニズム:** システム運用の全段階におけるコアとなる意思決定変数を、改ざん不可能な方法で静かに記録します。モデルのパラメータ選択、データの除外ルール、仮説検定条件の変更の追跡が含まれ、データ操作 (P-hacking) やAIのハルシネーションを根本的に排除します。

## 2. コア制御およびヒューマンコンピュータインタラクション層 (Control & HCI Layer)
本層は、人間の研究責任者 (Human PI) とAIシステム間の唯一の架け橋であり、自動化されたプロセスにおいて人間が常に最高の意思決定権と最終的な監督権を保持することを保証します。

* **2.1 ノイズ低減ダッシュボード (Human PI Dashboard)**
    * **機能定義:** 人間の研究者用の視覚的なコントロールコンソール。
    * **コアメカニズム:** 複雑な基礎となる技術的な推論プロセスを遮断し、重要なマイルストーン (ベースライン表の生成、コアモデルの収束など) のみを出力します。重大な論理的矛盾やデータの欠損によるブレークポイントに直面した場合、Human-in-the-loopの介入要求をトリガーし、人間のコマンドを受信して実行します。
* **2.2 プロジェクトマネージャー (PM) エージェント**
    * **機能定義:** システムの総司令官および論理解析のハブ。
    * **コアメカニズム:** 人間が入力した自然言語の研究意図を受け取り、それを**多次元の研究目的を含む研究ポートフォリオ (Research Portfolio) に核分裂させます**。複数の並行タスクの下方への分配を調整し、サブエージェントの進捗を監視し、学際的なエキスパートエージェント間で意見の相違が生じた場合の予備的な仲裁または問題のエスカレーションを実行します。

## 3. データ処理およびセキュリティゲートウェイ (Data Processing & Security Gateway)
本層は、すべての外部のマルチソース異種臨床データがシステムの中核となる分析エンジンに入る唯一のチャネルであり、データのコンプライアンスと高いS/N比を保証します。

* **3.1 コンプライアンスおよび匿名化モジュール (Compliance & Anonymization Module)**
    * **機能定義:** 医療データのセキュリティファイアウォール。
    * **コアメカニズム:** データの取り込み前に自動スキャンを実行し、保護対象保健情報 (PHI) を強制的に削除またはハッシュ化します。マルチセンターまたは地域をまたぐリアルワールドデータの呼び出しが、現地のプライバシー規制および倫理委員会の審査要件に準拠していることを保証します。
* **3.2 標準化およびクリーニングエージェント (Standardization & Cleaning Agent)**
    * **機能定義:** データエンジニアリングの実行者。
    * **コアメカニズム:** 事前に設定された臨床データベースの標準化フレームワーク (v4.0標準など) に厳密に従います。変数名辞書のマッピング、単位の統一、異常値のインターセプト、および横持ち/縦持ちテーブルの自動変換を実行し、同時にグローバルなデータインデックスディレクトリをリアルタイムで更新および維持します。

## 4. コア研究コラボレーションチーム層 (Core Research Team Layer)
本層は、高度に専門化された複数の大規模言語モデル (LLM) エージェントで構成され、人間の研究チームをシミュレートして学際的なコラボレーションを行い、具体的な研究タスクを実行します。

* **4.1 課題設計および文献エージェント (Design & Literature Agent)**
    * **機能定義:** 学術的背景の審査と実現可能性の評価を担当します。
    * **コアメカニズム:** PMエージェントによって発行された初期テーマに基づいて、セマンティックな文献検索を実施します。既存の研究の空白 (Research Gap) を正確に特定し、方法論的背景を生成し、テーマの革新性の評価を予測します。
* **4.2 統計方法論エージェント (Statistical Methodology Agent)**
    * **機能定義:** システムの数学およびアルゴリズム設計センター。
    * **コアメカニズム:** 臨床データの分布特性と研究目的に基づいて、最適な統計戦略を自動的に推論します。**並行戦略設計機能を備えており** (感度検証のために従来の多変量回帰とIPCWを同時に提案するなど)、コードは記述せず、厳密な数学的論理計画マトリックスのみを出力します。
* **4.3 コード開発エージェント (Code Development Agent)**
    * **機能定義:** 統計計画の正確なコード翻訳者。
    * **コアメカニズム:** 統計戦略マトリックスを受け取り、構文的に厳密で完全なコメントが付いた (PythonやR言語などの) 分析スクリプトを生成します。**複数モデルの並列環境分離コンピューティングを実行し**、出版レベルのデータチャートと多次元結果比較マトリックスを出力します。
* **4.4 集中治療臨床エキスパートエージェント (Clinical Expert Agent)**
    * **機能定義:** マクロな臨床的視点を提供する医学的解釈者。
    * **コアメカニズム:** 統計的有意性の結果を臨床実践の言語に翻訳します。現実世界の重症患者の蘇生、投薬ガイダンス、またはガイドラインの更新に対するデータ結果の実際の臨床的意義を評価し、研究が単なる統計的な数字ゲームに陥るのを防ぎます。
* **4.5 病態生理学エキスパートエージェント (Pathophysiology Expert Agent)**
    * **機能定義:** ミクロなメカニズムの視点を提供する理論的支持者。
    * **コアメカニズム:** 分子生物学、炎症経路、および血行動態の原理を組み合わせて、観察された臨床的表現型と統計的相関に対する深い病態生理学的因果メカニズムの説明を提供します。

## 5. 出力および査読層 (Output & Peer Review Layer)
本層は、研究成果を国際的な学術基準を満たす出版物に翻訳し、内部の「レッドチーム対ブルーチーム」の敵対的レビューを通じて非常に高い原稿の品質を確保する役割を担います。

* **5.1 学術執筆エージェント (Academic Writing Agent)**
    * **機能定義:** システムのチーフライターおよび組版エンジニア。
    * **コアメカニズム:** コードが出力したチャートと専門家の解釈ロジックを集約します。結果のボリュームに基づいてシステムレベルの組み立てを行い、**主分析を本文に、代替案/感度分析を補足資料に (または一連の論文に分割して) 記述します**。IMRAD構造に厳密に従って原稿を執筆し、高品質なLaTeX組版コードのネイティブ出力をサポートします。
* **5.2 厳格な査読者エージェント (Critical Reviewer Agent)**
    * **機能定義:** 内部品質管理の「レッドチーム」および脆弱性ハンター。
    * **コアメカニズム:** トップクラスの国際的ジャーナルの査読専門家の視点をシミュレートします。原稿内の方法論的欠陥、交絡因子の欠落、および結論の誇張のリスクを積極的に探し出し、鋭い却下コメントを生成して強制的な修正を促します。
* **5.3 編集長エージェント (Editor-in-Chief Agent)**
    * **機能定義:** 最終的な学術的基準の守護者。
    * **コアメカニズム:** 修正された最終原稿を総合的に評価します。論文の新規性、臨床的価値、および出版の可能性を比較検討し、シミュレートされた最終的な評決の結論を下します。
* **5.4 発散的思考エージェント (Divergent Thinking Agent)**
    * **機能定義:** 研究エコシステムの持続可能な進化を推進するシンクタンク。
    * **コアメカニズム:** 単一の研究のクローズドループ後、現在の研究の発見とこれまでの文献の矛盾、または本研究の限界に基づいて、その後の検証のためのまったく新しい仮説とエントリーポイントを自動的に推論および提案します。
